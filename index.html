让你的 golang 用上私有 mod

发表于 2020-08-10 | 更新于 2020-08-11 | 分类于 golang

复习 go mod 基础

先复习下 golang mod 的基础（仅列举出对本文章中使用到的命令，更多命令请参考官方文档）

  1. go mod init
    在当前目录新建 mod 文件，并启用 mod 进行依赖管理
  2. go mod tidy
    整理当前项目，添加未包含/删除无用的依赖
  3. go mod vendor
    将本地依赖添加到项目的 vendor 目录
  4. go mod edit
    编辑 mod 文件

对私有项目进行依赖的几种方法

1. 将私有项目拷贝到当前项目中（例如：pkg 目录中），替换掉项目中所有引用

优点:

  1. 不用考虑拉取依赖代码遇到的网络问题
  2. 私有项目改动带来的 bug 也无法带来影响
  3. 可以自由的对私有项目代码进行改动，不必担心影响其他人的集成

缺点：

  1. 想要使用最新的私有代码，只能手动复制替换掉原来的代码
  2. 调用私有代码出现问题，需要额外排查是否由于复制带来的问题
  3. 需要替换项目中已有的引用

2. 将私有项目拷贝到当前项目中（例如：pkg 目录中），使用 go mod edit -replace 指向本地目录

使用方式：

优点:

  1. 不用考虑拉取依赖代码遇到的网络问题
  2. 私有项目改动带来的 bug 也无法带来影响
  3. 可以自由的对私有项目代码进行改动，不必担心影响其他人的集成

缺点:

  1. 想要使用最新的私有代码，只能手动复制替换掉原来的代码
  2. 调用私有代码出现问题，需要额外排查是否由于复制带来的问题

3. [未验证]使用自建 goproxy

goproxy 可以代理各种开源依赖，并加速对依赖的访问，网上有许多对其进行改造以访问私有项目的案例。

优点:

  1. 加速所有依赖的访问速度
  2. 本地配置代理地址即可使用，无需改动项目任何代码
  3. 能随私有依赖的迭代而升级

缺点：

  1. 需要一台专门的服务器
  2. 需要改造，有额外的开发工作
  3. 使用需要进行网络请求，有连接失败的可能

4. [本文推荐]借助 gitlab Personal Access Tokens 访问私有依赖

gitlab Personal Access Tokens 提供 http 访问 gitlab 时认证的能力。既然私有依赖可以通过 http + 账号密码进行 git clone，那么通过 http + token 也可以进行访问。

优点：

  1. 仅在本地做一些配置即可访问私有依赖
  2. 依靠 go.mod，能主动选择使用的版本
  3. 不用对项目本身进行任何改动

缺点:

  1. 每位新加入的开发人员都需要做一定的配置
  2. 持续集成可能也需要进行一定配置

如何借助 gitlab Personal Access Tokens 访问私有依赖

1. 获取 gitlab token

  1. 进入Gitlab –> Settings –> Access Tokens # 可直接访问地址：{gitlab.example.com}/profile/personal_access_tokens
  2. 输入 name、过期时间、访问域（建议只选 read_repository）
  3. 点击生成

2. 将 token 配置在 git 请求头中

3. 将 git 对 ssh 的请求专为 http

4. 将私有仓库加入 GOPRIVATE，避免对其进行完整性校验

5. [可选]替换掉部分申明

6. 整理当前项目，添加需要的依赖

6. [可选]将已下载到本地的依赖打包进项目
